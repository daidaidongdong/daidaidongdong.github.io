hexo博客创建有感

hexo 搭建博客有感

  • 安装 hexo

    首先电脑上要有node还有git ，建议在桌面打开命行。

  • 初始化 hexo 博客项目
    1.在桌面新创建HEXO文件夹
    2.在HEXO文件夹内，右键运行Git Bash，输入命令:

    $ hexo init

    3.完成后将HEXO文件夹拖到VSCode中打开，会如下图文件。

  • 文档名介绍
    _config.yml 全局配置文件，网站的很多信息都在这里配置，诸如网站名称，副标题，描述，作者，语言，主题，部署等等参数, 这个文件下面会做较为详细的介绍 。
    scaffolds “ 脚手架、骨架 ”的意思，当你新建一篇文章（hexo new ‘title’）的时候，hexo是根据这个目录下的文件进行构建的 。

    source  这个目录很重要，新建的文章都是在保存在这个目录下的.
             _posts 。需要新建的博文都放在 _posts 目录下。
             _posts 目录下是一个个 markdown 文件。你应该可以看到一个 hello-world.md 的文 
             件，文章就在这个文件中编辑 。                                                          
             _posts 目录下的md文件，会被编译成html文件，放到 public （此文件现在应该没有，
             因为你还没有编译过）文件夹下。你可以用typora写好后将文件储存到source文件名下。
             还可以      ctrl + ~ 调出操作命令行
                               hexo new 输入博客名 （中文英文都可以） 
                               完成后在source文件目录下就会出现你的新建博客。
                               ctrl + s  保存

    themes 网站主题目录，hexo有非常好的主题拓展，支持的主题也很丰富。该目录下，每一个子目录就是一个主题 用法 ：去hexo官网的主题里面看别人的主题，进到别人的github上找到他的主题文件，克隆到themes文件夹内，点开在主目录下的_config.yml，在内容中找到theme 并将名字改成你下载的主题名字。

另外，点开你下载的主题，里面也会有_config.yml  在这里面你可以改掉别人主题里面的标题，网址等，还有source里面有用的图片，也可以改成你自己的图片。

  • 在本地中浏览博客
    用于在对博客有改动后（例如：换主题，写好了新博客要发布等）在本地试运行，运行成功后再放到浏览器上。ctrl+~调出命令行

    hexo clean
hexo g
hexo d

  • 部署到 Github 上
    1.申请Github账号 ( 别忘了邮箱验证 )
    2.新建一个 Repository , 注意仓库的名字前缀必须和自己 github 的用户名保持严格一致

    3.在_config.yml进行配置

    在 deploy 中配置自己的仓库信息 ( 注意冒号后面有空格 )改成自己的

    4.安装hexo-deployer-git自动部署发布工具

    $ npm install hexo-deployer-git -–save

    5.发布到Github

输入如下命令 :

$ hexo clean && hexo g && hexo d

然后输入自己的 Github 用户名和密码即可

6.在浏览器中输入 https://daidaidongdong.github.io便可以打开自己的博客啦 !!!

第4，5步是每一次更改后要发布到互联网上必须做的流程
